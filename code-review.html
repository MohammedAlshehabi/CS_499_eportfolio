Informal Code Review

This page provides a walkthrough of my original artifacts, analysis of areas for improvement, and my planned enhancements for CS 499 ePortfolio.

Existing Functionality

The original Contact Service application provided basic contact management with minimal validation and storage in memory. Algorithms used were simple linear searches and ArrayLists.

Code Analysis

  • Structure: Clear classes, but some duplicate code.
  • Efficiency: ArrayLists for lookups caused slow performance.
  • Security: No input validation or prepared statements.
  • Testing: Limited unit tests.

Planned Enhancements

  • Refactor code for maintainability and encapsulation (Outcome 2, 4)
  • Improve algorithms using HashMaps for efficiency (Outcome 3)
  • Add SQLite database for persistence (Outcome 3, 4, 5)
  • Implement input validation and prepared statements for security (Outcome 5)
  • Document process in narrative and ePortfolio (Outcome 2)
