Artifact Description

The artifact I selected for this enhancement is an improved Contact Service application that I originally created in CS 320. The original version of this application stored contacts in memory using an ArrayList. Contacts were lost when the application was closed and there was no persistence. The service allowed adding, retrieving, updating, and deleting contacts, but all data existed only while the program ran.

For Milestone Four, I enhanced this artifact to include a relational database using SQLite and JDBC. The enhancement added a persistent database layer so contacts are now stored in a database file and remain after the program closes. The application now demonstrates full CRUD functionality with a relational database schema.

Justification for Inclusion in the ePortfolio

I selected this artifact for my ePortfolio because it shows my ability to design and implement a database-backed application. The original Contact Service was a good starting point for this enhancement. It shows how I can move from basic in-memory data structures to real-world database persistence.

The enhanced artifact demonstrates the following skills:

  • Designing a relational database schema with constraints that enforce data integrity
  • Using SQLite as a lightweight, file-based database
  • Implementing a repository pattern to separate business logic from data access logic
  • Using prepared statements for all SQL operations to prevent SQL injection
  • Adding indexes to improve query performance
  • Writing database-backed tests that verify real CRUD behavior

These enhancements improved the application by replacing in-memory storage with a persistent database layer. This makes the software more scalable, maintainable, and professional.

Description of Enhancements

The main enhancement was adding SQLite and JDBC to store contacts in a database file. I created a table called Contacts with fields for contact ID, first name, last name, phone number, and address. I added constraints to ensure correct data lengths and types.

I created a ContactRepository interface with methods to create, find, update, and delete contacts. JdbcContactRepository implements this interface using prepared statements. The ContactService class now uses the repository instead of directly managing an ArrayList.

I also added indexes to improve query performance. Database-backed unit tests were written to verify that contacts are correctly created, retrieved, updated, and deleted. Proper exception handling and resource management were included to ensure the application works safely and reliably.

Reflection on the Enhancement Process
